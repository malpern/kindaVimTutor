## Canonical QA
Q: How do I jump back to where I was before a search?
A: kindaVim doesn't implement the jumplist. `Ctrl-O` and `Ctrl-I` (jump back / forward) don't work.
Unsupported: yes
TerminalVim: `Ctrl-O` jumps back through your jumplist (records from searches, `gg`, `G`, file edits, etc.). `Ctrl-I` (same as Tab) jumps forward. `:jumps` lists the history.
Related: / — search (kindaVim supports it, just no jump-back)

Q: Does `gd` go to the definition of a variable in kindaVim?
A: No — `gd` (go-to-local-definition) and `gD` (go-to-global-definition) rely on Vim's in-buffer search. They're unbound in kindaVim. For definition-jumping, use your host IDE's native shortcut (Cmd-click in Xcode, F12 in VS Code, etc.).
Unsupported: yes
TerminalVim: `gd` searches backward in the buffer for the first occurrence of the keyword under the cursor. `gD` goes to the first occurrence in the entire file.
Related: host IDE go-to-definition (Cmd-click, F12)

Q: How do I navigate back after jumping to a definition?
A: kindaVim has no jumplist. Use the host editor/IDE's back-navigation feature — Xcode has ⌃⌘-Left, VS Code has `Ctrl-minus`.
Unsupported: yes
TerminalVim: `Ctrl-O` goes back, `Ctrl-I` forward, through the jumplist.
Related: host-app back-navigation shortcut

Q: How do I jump to a tag in kindaVim?
A: Not supported — kindaVim doesn't integrate with ctags, LSP, or any tags file. Use the host IDE's symbol-navigation feature.
Unsupported: yes
TerminalVim: `Ctrl-]` follows the tag under the cursor. `Ctrl-T` jumps back through the tag stack. Requires a tags file generated by `ctags`.
Related: host IDE go-to-symbol
