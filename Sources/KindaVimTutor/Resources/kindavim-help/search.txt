## The Basics
- `/{pattern}` then Enter — search forward
- `?{pattern}` then Enter — search backward
- `n` — repeat the last search in the same direction
- `N` — repeat the last search in the opposite direction
- `*` — search forward for the word under the cursor
- `#` — search backward for the word under the cursor

## Canonical QA
Q: How do I search for text?
A: Press `/` in {{normal}}, type the text, press Enter. The cursor jumps to the first match forward of its current position. Press `n` to go to the next match, `N` to go to the previous.
Related: ? — search backward; n — next match; * — search word under cursor

Q: How do I search backward?
A: Press `?` instead of `/`, type your query, press Enter. Same semantics as `/` but in reverse. `n` and `N` follow the original direction (so after `?`, `n` still goes backward).
Related: / — forward search; N — reverse last search

Q: How do I search for the word my cursor is on?
A: Press `*` in {{normal}} to search forward for the exact word under the cursor, `#` to search backward. Matches whole words — "foo" won't match "foobar".
Related: g* — looser match (matches substrings); / — manual search

Q: What's the difference between `/` and `f`?
A: `/` searches the WHOLE document (and can span multiple lines). `f` only searches the current line and only for a single character. Use `f` for precise line-local jumps, `/` for finding text anywhere.
Related: f — line-local single char; n — repeat / search

Q: How do I repeat my last search?
A: Press `n` in {{normal}} to repeat in the original direction, `N` to reverse. If your last search was `/foo`, `n` goes forward; if it was `?foo`, `n` goes backward.
Related: * — search-word shortcut also sets the search; # — backward search word

Q: Is search case-sensitive?
A: By default yes — `/Foo` won't match "foo". Stock Vim has `:set ignorecase` and `:set smartcase` to change this, but kindaVim doesn't support Ex `:set` commands, so the default sensitivity is what you get.
Related: :set ignorecase — Ex command, not supported in kindaVim

Q: Does search wrap around the document?
A: Yes — hit the end and the search wraps to the beginning. You'll see a brief "search hit BOTTOM, continuing at TOP" indication in stock Vim; kindaVim's behavior inside native text fields follows the host app's search mechanics.
Related: n — keep going forward; ? — search other direction

Q: How do I search and replace?
A: kindaVim doesn't support `:s` (substitute). To replace: search with `/`, then `cgn` to change the next match — repeat with `.` to quickly change subsequent matches.
Related: :s — Ex substitute (not supported); gn — motion to next match; . — repeat last change

Q: How do I delete to the next match?
A: `d/pattern<Enter>` deletes from the cursor to the start of the next match. Useful for "delete everything up to X".
Related: c/pattern — change version; y/pattern — yank version

Q: What's the difference between `*` and `/word<Enter>`?
A: `*` does an automatic whole-word search for the word under the cursor — no typing required. `/word<Enter>` is manual and matches substrings. `*` is faster when you're already on the word you want to search for.
Related: g* — substring match version of *; # — backward

Q: How do I cancel a search I'm typing?
A: Press `Esc` or `Ctrl-C` during `/` input to cancel without searching. The cursor stays where it was.
Related: Enter — commit the search
