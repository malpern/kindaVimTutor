## Within the Line
- `0` — move to column zero (the very first character, even if it's whitespace)
- `^` — move to the first non-blank character of the line
- `$` — move to the end of the line

## Across the Document
- `gg` — jump to the first line
- `G` — jump to the last line
- `{count}G` — jump to a specific line number

## Canonical QA
Q: How do I jump to the start of a line?
A: Press `0` in {{normal}} to move to column 0 (the very first character). Press `^` instead if you want the first non-BLANK character — that's the more common "start of line" when a line is indented.
Related: $ — end of line; ^ — first non-blank

Q: What's the difference between `0` and `^`?
A: `0` goes to column 0 no matter what — if the line starts with spaces or tabs, you land on the whitespace. `^` skips leading whitespace and lands on the first real character. For indented code, `^` is almost always what you want.
Related: $ — end of line

Q: How do I jump to the end of a line?
A: Press `$` in {{normal}} to jump to the last character of the current line. If you want to append at the end, use the shortcut `A` — it jumps to end-of-line AND enters {{insert}} in one step (distinct from `$a` which lands after the last character).
Related: A — append at end of line; 0 — start of line

Q: How do I go to the top of the document?
A: Press `gg` in {{normal}} — two `g` taps — to jump to the first line.
Related: G — go to the last line

Q: How do I go to the bottom of the document?
A: Press `G` (capital) in {{normal}} to jump to the last line.
Related: gg — go to the first line; 10G — go to line 10

Q: How do I jump to a specific line number?
A: Type the line number followed by `G`. For example, `25G` jumps to line 25. For line 1, `1G` also works (same as `gg`).
Related: gg — first line; G — last line

Q: What does `G` without a count do?
A: Pressing `G` with no count jumps to the LAST line of the document.
Related: gg — first line; {count}G — specific line number

Q: Can I use these with operators?
A: Yes. `d$` deletes from the cursor to the end of the line. `d0` deletes to column 0. `dG` deletes from the current line to the end of the document. The same operator+motion grammar works everywhere.
Related: dd — delete the current line; D — same as d$

Q: How do I jump to column N on a line?
A: `|` (pipe) followed by a count moves to that column — e.g. `25|` jumps to column 25 on the current line.
Related: 0 — column zero; $ — last column

Q: How do I jump half a screen down?
A: kindaVim doesn't support `Ctrl-D` / `Ctrl-U` (half-page scroll) — those are tied to Vim's own viewport which kindaVim doesn't own. For jumping around a long document use `G`, `gg`, or a line-number jump like `100G`.
Unsupported: yes
TerminalVim: In stock Vim, `Ctrl-D` scrolls half a page down and `Ctrl-U` half a page up. `Ctrl-F` / `Ctrl-B` do full-page.
Related: G — last line; gg — first line; 100G — jump to line 100
