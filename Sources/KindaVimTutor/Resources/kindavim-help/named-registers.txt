## What Stock Vim Does
Vim has ~35 registers. `"{letter}y` yanks into the named register, `"{letter}p` pastes from it. `"+` is the system clipboard. `"0` always holds the last yank (even if you've deleted since). `"%` holds the current filename.

## What kindaVim Does
One default register. Each yank or delete replaces what's there. `y` and `p` are your tools; named-register prefixes do nothing.

For the system clipboard, use macOS-native `Cmd+C` / `Cmd+V` from {{insert}}.

## Canonical QA
Q: How do I yank into a named register like `"a`?
A: kindaVim doesn't support named registers. There's a single default register that `y` writes to and `p` reads from.
Unsupported: yes
TerminalVim: `"ay{motion}` yanks into register `a`. `"ap` pastes from it. Names range `a`-`z` (lowercase replaces, uppercase appends). View all with `:reg`.
Related: y — default yank; p — default put

Q: How do I copy to the system clipboard?
A: kindaVim doesn't support the `"+` register. Use macOS `Cmd+C` from the host app to copy to the system clipboard. Inside kindaVim's text, `y` yanks to kindaVim's internal register, NOT the clipboard.
Unsupported: yes
TerminalVim: `"+y` yanks to the system clipboard; `"+p` pastes from it. Requires Vim compiled with `+clipboard`.
Related: Cmd+C — macOS copy; Cmd+V — macOS paste

Q: Why doesn't `"ay` do anything?
A: kindaVim has no named registers, so the `"a` prefix is unbound — it's interpreted as "pending input, please continue typing the command." Once you press `y`, nothing happens because kindaVim doesn't know what `"ay` should do.
Unsupported: yes
TerminalVim: `"ay` works in stock Vim — yanks into register `a`. kindaVim's register model is deliberately simpler.
Related: y — default yank; p — default put

Q: How do I access the last yank after I've deleted something?
A: In stock Vim, `"0p` pastes from the yank-only register. kindaVim doesn't have numbered registers, so your last yank is overwritten by any delete. To preserve a yank, paste it somewhere first, then do the delete.
Unsupported: yes
TerminalVim: `"0p` pastes the LAST yank (never a delete). `"1p`-`"9p` paste recent deletes, shifting newer to lower numbers.
Related: y — yank (kindaVim); p — put

Q: How do I use the system clipboard from kindaVim?
A: Use the host app's `Cmd+C` / `Cmd+V` shortcuts. From {{insert}}, `Cmd+V` pastes the clipboard. kindaVim's `p` uses its own internal register separately.
Unsupported: yes
TerminalVim: `"+y` / `"+p` use the system clipboard directly. `"*y` / `"*p` use the selection primary (X11 style).
Related: Cmd+C — macOS copy; Cmd+V — macOS paste

Q: Can I see what's in the register?
A: kindaVim doesn't expose register contents — no `:reg` command. You'll know what's in it only by pasting to see.
Unsupported: yes
TerminalVim: `:reg` lists all register contents. `:reg a b +` shows specific ones.
Related: p — paste to see; P — paste before cursor
