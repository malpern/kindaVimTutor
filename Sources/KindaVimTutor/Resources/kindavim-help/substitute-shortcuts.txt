## Canonical QA
Q: Does `&` work in kindaVim?
A: No — `&` repeats the last `:s` substitution, and kindaVim doesn't implement Ex commands including `:s`. `&` is unbound.
Unsupported: yes
TerminalVim: `&` repeats the last `:s` substitution with the same pattern/replacement, on the current line. `g&` repeats it across the entire buffer.
Related: :s — Ex substitute (not supported); . — repeat last CHANGE (different concept)

Q: How do I repeat a find-and-replace in kindaVim?
A: kindaVim doesn't support `&`, `g&`, or `:s`. The workaround: search with `/pattern<Enter>`, manually change the first match with an appropriate change operator (e.g. `cw` + replacement + `Esc`), then alternate `n` (next match) and `.` (repeat change).
Unsupported: yes
TerminalVim: `&` repeats the last `:s` on the current line; `g&` repeats it across the whole buffer. kindaVim doesn't implement either because `:s` isn't available.
Related: / — search (supported); . — repeat change; cw — change word

Q: What's the difference between `.` and `&`?
A: `.` repeats the last CHANGE (insert, delete, etc.) and IS supported in kindaVim. `&` repeats the last `:s` substitution and is NOT supported. For the search-and-change workflow use `.` — it covers most of what `&` would do.
Unsupported: yes
TerminalVim: In stock Vim, `.` and `&` coexist — `&` is specifically for replaying `:s` with the same pattern, while `.` replays any change.
Related: . — repeat change (supported); :s — substitute (not supported)

Q: Can I repeat a substitution across multiple lines in kindaVim?
A: Not via `:s` — that's the unsupported Ex mechanism. The kindaVim workflow is `/pattern<Enter>`, manually change the first match (e.g. `cw<replacement><Esc>`), then `n.` (next match, repeat change) to walk down the document.
Unsupported: yes
TerminalVim: `:%s/old/new/g` hits every match in the buffer. `:%s/old/new/gc` asks for confirmation per match.
Related: / — forward search (supported); . — repeat change
